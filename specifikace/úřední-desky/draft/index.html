Úvod

Tento dokument popisuje datový standard, jak zveřejňovat informace o oznámeních z úředních desek jako otevřená data. Ukazuje, jak reprezentovat jednotlivé informace o platnosti oznámení, obsahu, odkazy na konkrétní dokumenty, apod.

Data mohou být v různých formátech - příklady jsou zde v JSON, XML a CSV.

Reprezentace v otevřených formátech

Soubory s datovými schématy si můžete stáhnout, zahrnout do svých publikačních plánů a upravit dle vlastních potřeb.

JSON

Schéma pro formát JSON (JSON Schema)

XML

Schéma pro formát XML (XSD, XML Schema)

CSV

Je také možné použít zjednodušený zápis v CSV. Má ale jistá omezení. Je možné použít max. 1 prvek, který v obecném formátu může mít více položek (např. další soubory/přílohy oznámení, agendy). Také nelze používat vlastní atributy nad rámec standardu.

Proč by nešlo použít vlastní atributy? to lze vždy

K dispozici je Datové schéma dle specifikace CSV on the Web.

Jména sloupců se tvoří z atributů, zanořování se řeší spojením názvů pomocí "_".

Příklady

Příklady jsou zde uvedené nejprve zapsané v JSON. Dále potom ekvivaletně v XML a ukázáno, jak některé zapsat v CSV.

Jednoduché oznámení

Všechny atributy schématu jsou nepovinné. Nicméně bez některých základních atributů publikace prakticky ztrácí smysl.

Příklad minimalistického oznámení zveřejněného na úřední desce, odpovídá prvnímu diagramu:

Zápis v JSON: 

			[
			    {
			        "id": 1024,
			        "identifikator": "2018-13",
			        "organizace": {
			            "ico": "00258245"
			        },
			        "url": "https://example.com/uredni_deska/2018-2",
			        "nazev": "Podpora spolkového života ve městě",
			        "zverejneno": "2018-01-20",
			        "sejmuto": "2018-02-20",
			    }
			]

Zápis v XML: 

			<?xml version="1.0" encoding="UTF-8" ?>
			<uredni_deska>
			    <oznameni>
			        <id>1024</id>
			        <identifikator>2018-13</identifikator>
			        <organizace>
			            <ico>00258245</ico>
			        </organizace>
			        <url>https://example.com/uredni_deska/2018-2</url>
			        <nazev>Podpora spolkového života ve městě</nazev>
			        <zverejneno>2018-01-20</zverejneno>
			        <sejmuto>2018-02-20</sejmuto>
			    </oznameni>
			</uredni_deska>

Jednoduché oznámení s vlastními atributy

Do schématu si lze přidávat libovolně další vlastní atributy.

Příklad přidání vlastních atributů nad rámec standardu (zde zverejnil a odbor):

Zápis v JSON: 

			[
			    {
			        "id": 1025,
			        "identifikator": "2018-14",
			        "organizace": {
			            "ico": "00258245"
			        },
			        "url": "https://example.com/uredni_deska/2018-3",
			        "nazev": "Podpora spolkového života ve městě II",
			        "zverejneno": "2018-01-20",
			        "sejmuto": "2018-02-20",
			        "zverejnil": "Mirek Dušín",
			        "odbor": "VD"
			    }
			]

Zápis v XML, vlastní atributy se přidávají na závěr oznámení. Je také třeba nadefinovat vlastní namespace. 

			<?xml version="1.0" encoding="UTF-8" ?>
			<uredni_deska xmlns:x="https://example.com">
			    <oznameni>
			        <id>1025</id>
			        <identifikator>2018-14</identifikator>
			        <organizace>
			            <ico>00258245</ico>
			        </organizace>
			        <url>https://example.com/uredni_deska/2018-3</url>
			        <nazev>Podpora spolkového života ve městě II</nazev>
			        <zverejneno>2018-01-20</zverejneno>
			        <sejmuto>2018-02-20</sejmuto>
			        <x:zverejnil>Mirek Dušín</x:zverejnil>
			        <x:odbor>VD</x:odbor>
			    </oznameni>
			</uredni_deska>
